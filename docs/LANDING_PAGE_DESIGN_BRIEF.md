# Groove Metronome Landing Page Design Brief

## Project

Groove Metronome by Nathaniel School of Music

Tagline options:
- See it. Count it. Lock it in.
- A serious metronome for musicians who actually practice.
- Visual rhythm, stage-safe tempo, musical click sounds.

## Sales Objective

Design a premium, advertisable landing page that converts musicians, teachers, students, bands, and producers into users of Groove Metronome across:

- Web Browser app
- iOS / iPadOS app
- Android app
- DAW Plugin: AU, VST3, and later AAX

The page should make the product feel professional enough for serious musicians, simple enough for students, and rugged enough for stage and studio use.

Primary CTA:
- Try Groove Metronome

Secondary CTAs:
- Download for iOS
- Use in Browser
- Join Plugin Waitlist
- Share with Band

## Product Positioning

Groove Metronome is not just a basic click. It is a musician-first rhythm practice and performance tool with visual beat maps, per-beat subdivisions, polyrhythm, polymeter, setlists, audio/MIDI analysis, concert mode, and musical click sounds.

Position it against ordinary metronome apps as:

- More visual
- More musical
- More stage-ready
- Better for teachers and students
- More advanced without feeling complicated
- Useful across phone, iPad, browser, and eventually DAWs

## Audience

Primary users:
- Music students
- Drum, guitar, piano, vocal, and bass students
- Teachers
- Band leaders
- Drummers on stage
- Producers and DAW users
- Classical, jazz, rock/pop, Hindustani, and Carnatic musicians

Emotional need:
- They want rhythm to feel clear, visible, musical, and trustworthy.
- They do not want a sterile utility app that feels like a calculator.
- On stage, they need big controls, safety, and confidence.
- In practice, they need structure and motivation.

## Visual Direction

Design should feel:
- Premium
- Sleek
- Musical
- Fast
- Modern
- Stage-readable
- iPad and phone-friendly

Avoid:
- Brown-heavy palettes
- Cheap neon DJ visuals
- Generic SaaS cards everywhere
- Cute toy-like music graphics
- Overexplaining technical implementation
- Backend words like VexFlow, Tone.js, Capacitor, etc.

Use:
- Strong rhythm visuals
- Large app screenshots/mockups
- Clean black/dark performance background
- Accent colors from current app themes
- Elegant notation fragments
- Subtle motion language: pulse, lock, beat, stage
- Real NSM logo and Groove Metronome logo system

## Required Page Sections

### 1. Hero

Goal: immediate clarity and desire.

Content:
- Groove Metronome
- See it. Count it. Lock it in.
- A visual metronome for practice, stage, and studio.

CTAs:
- Try Web App
- Get iOS App
- Plugin Waitlist

Hero visual:
- Large phone + iPad + laptop layout showing the app
- Show beat wheel / setlist / analyzer subtly
- Make it obvious this is a real product, not a concept

### 2. Platform Strip

Show all versions:

- Web Browser: works instantly
- iPhone / iPad: touch-first practice and stage mode
- Android: mobile rhythm practice
- DAW Plugin: AU / VST3 / AAX planned for studio use

Use clear platform badges, not clutter.

### 3. Why It Is Different

Short value blocks:

- Visual beat map
- Per-beat subdivisions
- Musical click sounds
- Polyrhythm and polymeter
- Setlist Studio for concerts
- Audio/MIDI analyzer
- Offline-ready performance mode

Use compact, musician-facing language.

### 4. Practice Modes

Show the main practice surfaces:

- Beat Map
- Levels
- Polyrhythm
- Polymeter
- Rhythm Assist
- Practice Ramp
- Mute Trainer

The page should make these look approachable, not intimidating.

### 5. Setlist Studio / Concert Mode

This should be a major sales section.

Message:
- Build your show.
- Reorder songs.
- Copy and edit songs.
- Back up and share setlists.
- Panic recover if something goes wrong.
- Big stage-safe controls.

Show:
- Song list
- Current song
- Big tempo
- Clock/concert time
- Panic recover button

### 6. Musical Assist Packs

Show that this app respects real musical worlds:

- Jazz: 2 and 4, Charleston, swing ands, ride feel
- Hindustani: Teentaal, Keherva, Dadra
- Carnatic: Adi Tala, Misra Chapu, Konnakol patterns
- Classical: tempo ladder and subdivision clarity
- Rock/Pop: backbeat and groove practice

Keep this tasteful. Do not overclaim full tabla/iTabla replacement. Phrase as practice references and rhythm assist packs.

### 7. Analyzer

Position as:
- Drop audio or MIDI
- Detect tempo
- Play with metronome
- Review sections and markers
- Use detected tempo in the metronome

Avoid claiming perfect automatic time-signature detection.

### 8. Sounds

Message:
- Musical click sounds, not harsh beeps.
- Marimba default
- Wood, clave, shaker, tabla, rim, tight, voice count-ins
- Smart sound matching for subdivisions and assist layers

Mention voice packs as studio-recordable / coming soon if needed.

### 9. DAW Plugin

Status wording:
- DAW Plugin in development
- AU and VST3 first
- AAX planned after the core plugin is stable
- Designed for Logic, Reaper, Ableton, Cubase, Studio One, and Pro Tools

Plugin promise:
- Host tempo sync
- Musical click sounds
- Accents and subdivisions
- Voice count-in
- Studio-safe click engine

CTA:
- Join Plugin Waitlist
- Request Beta Access

Do not imply the plugin is available today unless it is actually shipped.

### 10. Teacher / School Use

Message:
- Great for teachers assigning rhythm practice
- Useful in online classes
- Students can share setlists or practice setups
- Visual rhythm helps beginners understand subdivisions

### 11. Social Proof / Trust

Use placeholders if real testimonials are not ready:

- Designed for practice rooms, classrooms, and stage use

Later add:
- Student quotes

### 12. Final CTA

Keep it simple:

- Start practicing with Groove Metronome
- Try Web App
- Get iOS App
- Join Plugin Waitlist
